fputc

返回值：

上面是我们在linux平台在帮助手册里面查看到的。
接下来我们给出代码说明：

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>

//通过fopen打开一个文件  返回一个FILE * 指针，以后我们所有对于文件的操作都是通过FILE * 指针

int main()
{
    FILE* pf = fopen("xx.txt","w+");
    if (pf == NULL)
    {
        printf("open error\n");
        exit(-1);
    }
    if (fputc('a', pf) == EOF)
    {
        printf("write error\n");
    }
    for ( char ch = 'a';ch  <= 'z';  ch++)
    {
        fputc(ch,pf);
    }
    fclose(pf);
    return 0;
}

上面代码执行结果为：

我们可以看到写入成功。

但是一般情况下来说写的过程不会出现问题，在代码中我们给出了写入是否成功的判断，我们可以看到：

如果写入错误就会返回EOF如果写入成功就会返回写入的数据。

fgetc

函数返回值：

在这里我们加入一个带参数的宏定义，因为我们每次都需要判断文件打开是否成功，所以我们把判断文件打开是否成功写入一个宏，便于我们判断方便而且降低代码的冗余。
我们通过一下方法：

#define F_PRINTF_ERR(e)\
do\
{\
    if (e == NULL)\
    {\
        printf("open error\n");\
        exit(0);\
    }\
} while (0)\

其中\代表续航符 上面整体代码表示一个的宏定义，然后通过个do，while进行判断，当然也可以用其他方式进行判断。我们可以观察到，while的后面没有;在实际的代码中我们刚好需要在一条语句执行完成之后加上;所以刚好和这里的宏定义组成规则的do，while语句。那么定义好了之后我们就可以在其他需要判断文件是否打开成功的时候直接使用带参数的宏定义。

我们给出代码：

#define _CRT_SECURE_NO_WARNINGS

#include <stdlib.h>
#define F_PRINTF_ERR(e)\
do\
{\
    if (e == NULL)\
    {\
        printf("open error\n");\
        exit(0);\
    }\
} while (0)\



int main()
{
    FILE* pf = fopen("asd.txt", "r");

    //{
    //    printf("open error\n");
    //    exit(-1);

执行结果为：

中间注释的部分是我们之前判断文件是否打开成功的方法，现在我们写入宏定义里面之后我们就可以通过定义的宏来直接判断。

那么现在我们有一个问题就是，我们在读取的时候读取到什么时候结束呢？我们读取数组需要知道三要素，我们读取字符串找‘\0’结束，那么读取文件的结束位置也同样是很重要的。
我们首先把文件里面所包含的内容读取出来：我们刚开始保存了26个英文字母。

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>
#define F_PRINTF_ERR(e)\
do\
{\
    if (e == NULL)\
    {\
        printf("open error\n");\
        exit(0);\
    }\
} while (0)\



int main()
{
    FILE* pf = fopen("xx.txt", "r");
    //if (pf == NULL)
    //{
    //    printf("open error\n");
    //    exit(-1);
    //}
    F_PRINTF_ERR(pf);
    for (int i = 0; i < 26; i++)
    {
        if (i % 5 == 0)
            printf("\n");
        putchar(fgetc(pf));
        printf("\t");
    }

    fclose(pf);
    return 0;
}

执行结果为：

那么我们是刚好给文件存了26个字符，我们自己规定读取到了26个字符。那么在一个文件中我们自己有时候是没法确定文件的位置的，如果文件很长的话，我们能够一个一个去数吗？
例如：

do\

执行结果为：

我们可以看到，读取到了很多不可打印的字符。
为我们换一种方式，之前打印30个字符，现在我们打印这30个字符的地址：

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>
#define F_PRINTF_ERR(e)\
do\
{\
    if (e == NULL)\
    {\
        printf("open error\n");\
        exit(0);\
    }\
} while (0)\



int main()
{
    char ch;
    FILE* pf = fopen("xx.txt", "r");
    //if (pf == NULL)
    //{
    //    printf("open error\n");
    //    exit(-1);
    //}
    F_PRINTF_ERR(pf);
    for (int i = 0; i < 30; i++)
    {
        if (i % 5 == 0)
            printf("\n");
        ch = fgetc(pf);
        putchar(ch);
        printf("\t%p\t",ch);
    }

    fclose(pf);
    return 0;
}

执行结果为：

我们可以看到，前面的26个字母我们所读取到的数据和地址都没有问题。但是后面的4个读取到了什么呢？ 其实我们之后的4个读取到的是EOF EOF 的本质就是-1。所以我们必须有一个标准来作为我们文件读取完的标志：所以我们用while循环实现

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>
#define F_PRINTF_ERR(e)\
do\
{\
    if (e == NULL)\
    {\
        printf("open error\n");\
        exit(0);\
    }\
} while (0)\



int main()
{
    char ch;
    FILE* pf = fopen("xx.txt", "r");
    //if (pf == NULL)
    //{
    //    printf("open error\n");
    //    exit(-1);
    //}
    F_PRINTF_ERR(pf);
    while ((ch = fgetc(pf)) != EOF)   //注意运算符优先级
    {
        putchar(ch);
        printf("\t%p\n", ch);
    }

    fclose(pf);
    return 0;
}

执行结果为：

这样我们就可以很放心的来直接读取。

feof

feof 这个函数，是去读标志位判断文件是否结束的。即在读到文件结尾的时候再去读一次，标志位才会置位，此时再来作判断文件处理结束状态，文件到结尾。如果用于打印，则会出现多打一次的现象

我们先来看不多打印的一次情况，然后进行对比分析理解。
代码演示：

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>
//下面的 \ 符号称为续航符
#define F_PRINT_ERR(e)\
do\
{\
if (e == NULL)\
{\
        printf("open error\n");\
        exit(-1);\
}\
}\
while (0)\

//通过fopen打开一个文件  返回一个FILE * 指针，以后我们所有对于文件的操作都是通过FILE * 指针




int main()
{
    FILE* pf = fopen("xx.txt", "w+");
    F_PRINT_ERR(pf);
    for (char ch = 'a'; ch <= 'z'; ch++)
    {
        putchar(fputc(ch, pf));
    }
    rewind(pf);
    char data;
    printf("\n==========================\n");
    while ((data = fgetc(pf)) != EOF)
    {
        printf("%c", data);
    }
    fclose(pf);
    return 0;
}

我们可以清楚的看到使用feof会在判断判断文件是否读到文件结尾之后再多读一次字符，导致的原因就是标志位检测滞后所导致的。
