什么是行

定义
行是文本编辑器中的概念，文件流中就是一个字符。这个在不同的平台是有差异的。window 平台 ‘\r\n’，linux 平台是’\n’。

我们很容易理解，在文本文件里面这就是两行：

我们可以看到文本文件会给我们呈现出所谓的行的概念，但是如果用二进制形式打开的话，里面就是一个控制字符。

我们使用二进制文件进行查看：
我们通过hexdump -C line.txt 命令进行二进制查看：

文本文件的本质是呈现给人来看的，我们可看到中间有0a，0a就是换行，在linux平台我们可以看到中间有一个0a 在结束的时候也会自动加一个0a，最后加0a会给我们带来很多便利。

平台差异

windows 平台在写入’\n’是会体现为’\r\n’，linux 平台在写入’\n’时会体现为’\n’。

windows 平台在读入’\r\n’时，体现为一个字符’\n’，linux 平台在读入’\n’时，体现为 一个字符’\n’。

在windows二进制体现为OD OA

对应ACSII:OD（13）代表回车 （10）OA代表换行

回车在windows文本里面表把光标移动到头部，然后再通过OA换到下一行。

也就是说在windows中 ‘\r\n’ 前面\r表示回车，后面\n表示换行，两个字符在一起所体现的功能和linux中的’\n’是一样的。

所以如果你把已经换行的linux平台的文本移动到windows平台是不会出现换行的。因为文本编译器发现没有OD OA 所以无法换行。

linux 读 windows 中的换行，则会多读一个字符，windows 读 linux 中的换行，则没有问题。会直接把linux中的一个换行用一个字符表示方法来读取，我们后面会有代码演示。

我们前面进行过说明：

windows中 ‘\r\n’ 前面\r表示回车，后面\n表示换行，两个字符在一起所体现的功能和linux中的’\n’是一样的。也就是说windows平台的两个字符和linux平台的一个字符的作用是一样的，并且在写入操作的时候，在windows平台我们在代码中使用fputc(‘\n’, fp);写入’\n’一个字符也没有问题，但是真正在进行读写操作的时候，在window平台读写过程中也是把’\r\n’ 合并为一个’\n’进行操作所以我们在windows平台写入的时候也是直接使用fputc(‘\n’, fp); 读取的时候直接读取也是一个字符。所以读取文件三次就可以读完，因为中间的换行符整体操作。这样在写代码的过程中也就屏蔽了平台的差别。

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>
#define F_PRINTF_ERR(e)\
do\
{\
    if (e == NULL)\
    {\
        printf("open error\n");\
        exit(0);\
    }\
} while (0)\

int main()
{
    FILE* pf = fopen("linux.txt", "r+");
    F_PRINTF_ERR(pf);
    char ch;
    ch = fgetc(pf);
    printf("%x",ch);
    ch = fgetc(pf);
    printf("%x", ch);
    ch = fgetc(pf);
    printf("%x", ch);
    ch = fgetc(pf);
    printf("%x", ch);
    fclose(pf);
    return 0;
}

小结

不同的平台在开发的时候同样的内容定义的不同，我们必须要提前了解到不同之处并且做出处理才能避免在跨平台编程的时候出现一些因为平台差异性而出现的一些问题，所以我们在开发过程中跨平台开发的时候就需要做一些处理。
