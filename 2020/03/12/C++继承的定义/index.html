继承的定义

类的继承，是新的类从已有类那里得到已有的特性。已有类产生新类的过程就是类的派生。原有的类称为基类或父类，产生的新类称为派生类或子类。
派生与继承，是同一种意义两种称谓。

我们已经在继承引入中详细说明。
从已有的父类中进行继承的时候，父类相当于是共性，例如我们给出的Human类，子类通过继承得到父类的特性，但是每一个子类都有自己个性只属于每一个子类。

定性 is-a Not has-a

is-a 是一种 属于关系。
如：狗属于一种动物，车属于一种交通工具(Dog is an Animal. Car is a Vehicle.)在面向对象中表现为一种继承关系。可以设计一个 Animal 类，Dog 类作为Animal 类(基类)的派生类；设计一个 Vehicle 类，Car 类作为 Vehicle 类(基类)的派生类。

has-a 是一种 包含、 、 组合关系。
如：车包含方向盘、轮胎、发动机(Car has steering-wheel,wheels, engine)，但不能说方向盘/轮胎/发动机是一种车；狗包含腿、尾巴，但不能说腿、尾巴是一种狗。正确的应该说车聚合(包含)了方向盘、轮胎、发动机。

因此，如果 A is a B，则 B 是 A 的基类，A 是 B 的派生类。为继承关系。如果 A 包含 B，则 B 是 A 的组成部分。为聚合关系，可以由组成部分聚合成为一个类。

宏观意义上来讲， is-a 和 has-a 均可以实现代码的可重用性。

语法

class 派生类名：[继承方式] 基类名
{
派生类成员声明；
};

默认的继承方式是 private 私有继承。

一个派生类可以同时有多个基类，这种情况称为多重继承，派生类只有一个基类，称为单继承。

继承方式

分类

我们先来说明public公有继承。

继承方式影响了什么？

继承方式规定了子类如何访问从基类继承的成员。

继承方式有 public, private, protected。继承方式不影响派生类的访问权限， 影响了从基类继承而来的成员的访问权限，包括派生类内的访问权限和派生类对象的访问权限。

protected 在外部对象访问的时候，等价于private
在类内所有的访问权限都是公开的。

也就是说不影响派生类在类内部成员的访问方式，影响了基类成员在派生类内部和派生类创建的外部对象的访问权限。

我们再来看一下共有类public的继承方式：
基类的公有成员和保护成员在派生类中保持原有访问属性，其私有成员仍为基类的私有成员。派生类中没有访问权限

#include <iostream>

//继承方式影响了什么？不影响什么？

class A
{
public:
int pub;
protected:
int pro;
private:
int pri;
};

class B : public A
{
public:
    int pubb;
protected:
    int prob;
private:
    int prib;
};

int main()
{
B b;
b.pubb;
b.prob;
b.prib;
return 0;
}

上面代码中：

b.prob;
b.prib;

会报错。

上面的定义方式不管B类对于A类是否继承，都不会影响到B类在类内对于所有成员的访问。

我们再来看下面代码：

#include <iostream>

using namespace std;

//继承方式影响了什么？不影响什么？

class A
{
public:
int pub;
protected:
int pro;
private:
int pri;
};

class B : public A
{
public:
void func()
{
cout << pub << endl;
cout << pro << endl   //protected继承自后在子类可见
cout << pri << endl; //不可见，无法访问
}
    int pubb;
protected:
    int prob;
private:
    int prib;
};

int main()
{
B b;
b.pub;
b.pro;  //出错
b.pri;  //出错
return 0;
}

继承之后父类的成员在子类内部可以访问是public成员和protected成员，父类中的private成员在子类的内部不允许访问。

b.pro;  //出错
b.pri;  //出错

两行代码出错
说明父类的public成员在子类继承之后创建的外部对象可以访问。

父类的private成员和protected成员在子类继承之后创建的外部对象没有访问权限。

子类从父类继承public成员之后在子类中仍然为public。所以子类的对象可以访问。

子类从父类继承private成员之后在子类中直接不可访问，所以子类创建的对象也就不会有访问权限。

但如上面代码中，子类在继承父类的protected的时候，我们现在只是知道了在子类中可以访问，在子类创建的对象中没有访问权限我们并不清楚。所以子类对于从父类继承下来的protected到底是protected还是private我们需要验证：
我们再写一个C类进行继承B类：

class A
{
public:
int pub;
protected:
int pro;
private:
int pri;
};

class B : public A
{
public:
void func()
{
cout << pub << endl;
cout << pro << endl
cout << pri << endl; //不可见，无法访问
}
    int pubb;
protected:
    int prob;
private:
    int prib;
};

class C : public B
{
public:
    void foo()
    {
        cout << pub << endl;
        cout << pro << endl
        cout << pri << endl; //不可见，无法访问
    }
    int pubb;
protected:
    int prob;
private:
    int prib;
};

C类中的cout << pro << endl； 通过了
所以子类从父类继承protected成员之后类内仍可见，所以在子类C中pro仍然为protected。

所以继承方式中，父类的protected成员在子类继承之后仍为protected。

protected 在外部访问的时候，等价于private。所以子类的对象不具有访问权限。

派生类的组成

计算类的大小

#include <iostream>

using namespace std;

//继承方式影响了什么？不影响什么？

class A
{
public:
int pub;
protected:
int pro;
private:
int pri;
};

class B : public A
{
public:
    int pubb;
protected:
    int prob;
private:
    int prib;
};

class C : public B
{
public:
    int pubb;
protected:
    int prob;
private:
    int prib;
};

int main()
{
    cout << sizeof(A) << endl;
    cout << sizeof(B) << endl;
return 0;
}

运行结果为：

我们可以看到，子类会继承父类的所有成员。
包括public , protected ,private 所有的成员，即使private中的数据成员继承之后在类中没有访问权限，也会继承。

继承往往是因为需要的功能，如果是子类中已有的功能再进行继承的话就会出现派生类成员冗余，所以基类需要设计，避免派生类的冗余。

组成图示

派生类中的成员，包含两大部分，一类是从基类继承过来的，一类是自己增加的成员。

从基类继承过来的表现其共性，而新增的成员体现了其个性。

所有的this指针都是指向生成的对象d。

内存模型

三个类构造器的this指针指向的是同一个地址，也就是子类C生成对象的地址。
我们也容易理解，类是不会有地址的，对象才有地址，那么继承之后的子类C只生成了一个对象，那么这三个类中构造器的this指针都是子类C生成对象的地址。

但是这几个this是有区别的，三个构造器中的地址相同，类型不同。

代码演示：

#include <iostream>
#include <typeinfo>
using namespace std;


class A
{
public:
    A()
    {
        cout << "A()" << endl;
        cout << "&a:" << &a << endl;
        cout << "A-this:" << this << endl;
        cout << typeid(this).name() << endl;
    }
    int a;

};

class B : public A
{
public:
    B()
    {
        cout << "B()" << endl;
        cout << "&b:" << &b << endl;
        cout << "B-this:" << this << endl;
        cout << typeid(this).name() << endl;
    }
    int b;
};

class C : public B
{
public:
    C()
    {
        cout << "C()" << endl;
        cout << "&c:" << &c << endl;
        cout << "C-this:" << this << endl;
        cout << typeid(this).name() << endl;
    }
    int c;
};

int main()
{
    C d;
    cout << "&d:" << &d << endl;
    return 0;
}

运行结果为：

上面代码也是证明派生类在继承的过程中基类的成员全部继承。
我们可以看到打印出来 class A * 等说明他们的类型不同。

我们这里只是引入这种方式来说明类型不同，关于函数用法这里不做深究。
