派生类的构造

派生的过程不管子类有没有需要，都是把基类的所有成员全部继承。全部继承就会出现问题。
基类的部分成员在派生类中是不可见的，不可见就没有办法在派生类里面进行初始化，那么在派生类中不可见的继承部分就需要在基类的构造器中进行初始化，但是派生类在继承基类的时候会继承除了构造器和析构器之外的其他所有成员，所以我们在这里不能直接调用父类的构造器进行初始化，那么基类的构造器没有被派生类继承，派生类就无法直接调用基类的构造器对于派生类继承基类之后不可访问的成员进行初始化。

派生类中，从基类继承而来的成员的初始化工作还是由基类的构造器完成，然后派生类中原有的成员在派生类的构造函数中初始化。所以实际上并非是直接调用基类的构造器。

子类的构造器调用父类的构造器的情况分析

父类和子类都是默认无参构造器

代码演示:

#include <iostream>
using namespace std;

class A
{
public:
    A()
    {
        cout << "A()" << endl;
    }
private:

};

class B : public A
{
public:
    B()
    {
        cout << "B()" << endl;
    }
};


int main()
{
    B b;
    return 0;
}

由结果我们得到父类和子类都是无参构造器时生成对象会先默认调用A的构造器再默认调用B的构造器。

子类为无参默认构造器，父类为带参数的构造器且有默认值

当子类使用的是无参构造器的时候，会调用父类的默认无参构造器，如果父类构造器自实现，父类的构造器有参数并且有默认值，则父类的成员通过父类的构造器进行初始化。

显式调用父类构造

上面子类对于父类的调用，无论哪一种情况都无法把通过子类创建对象的默认参数传递给父类进行初始化。

语法格式

派生类名::派生类名(总参列表)
:基类名(参数表),内嵌子对象(参数表)
{
派生类新增成员的初始化语句; //也可出现地参数列表中
}

如果不是上面的标配情况就必须显式调用。

显式调用不管有没有标配都是适用的。

显式调用代码演示

#include <iostream>
using namespace std;

class A
{
public:
    A(int y)
    {
        a = y;
        cout << "A()" << endl;
        cout << "a = " << a << endl;


    }
    int a;
private:

};

class B : public A
{
public:
    B(int x, int y)
        :A(y)//显式调用
    {
        b = x;
        cout << "B()" << endl;
        cout << "b = " << b << endl;
    }
    int b;
};


int main()
{
    B c(10, 20);
    cout << "c.a = " << c.a << endl;
    cout << "c.b = " << c.b << endl;
    return 0;
}

那么在调用的时候符合标配情况进行隐式调用也可以直接使用显式调用。

小结

子类中包含两个部分，一部分来自于父类，一部分是子类自己有的。父类的通过父类构造器进行初始化，子类的通过子类构造器进行初始化。在初始化的过程中由于父类的构造器没有被子类继承，所以不能像调用函数一样调用父类的构造器，只能把父类的构造器通过上面语法格式中：

基类名（参数列表）

通过传参的方式进行初始化。

子类当中直接使用子类的构造器进行初始化。

构造顺序

图示

内嵌子对象

#include <iostream>
using namespace std;


class C
{
public:
    C()
    {
        cout << "C()" << endl;

    }
    int c;
private:
};


class A
{
public:
    A(int i)
    {
        cout << "A()" << endl;
        a = i;
    }
    int a;
private:

};

class B : public A
{
public:
    B(int i,int j)
        :A(i)
    {
        cout << "B()" << endl;
        b = j;
    }
    int b;
    C c;  //内嵌子对象
};


int main()
{
    B c(3,5);
    cout << c.a << endl;
    cout << c.b << endl;
    return 0;
}

运行结果为：

通过运行结果我们也可以看到，先默认调用父类构造器初始化父类，再默认调用内嵌类构造器初始化内嵌子类，最后默认调用子类构造器初始化子类。

上面的过程类似于我们在家招待亲戚，例如倒酒的时候，先给父亲倒上，再给亲戚倒上，最后给自己倒上。

调用内嵌子对象的构造器，标配会被调用，没有标配，必须显式调用。所以在调用的时候如果隐士的调用，就必须有标配，如果显示的构造，是否标配都不会影响。

那么内嵌子对象标配的情况和我们上面子类构造器中调用父类的构造器三种情况相同。

如果是非标配，则需要使用显示调用。
代码演示：

总结

由于子类中，包含了两部分内容，一部分来自父类，一部分来自子类。父类的部分，要由调用父类的构造器来完成，子类的部分，在子类的构造器中来完成始化。子类中有内嵌的子对象也需要构造。

简单应用

#include <iostream>

using namespace std;


class Brithday
{
public:
Brithday(int y, int m, int d)
:year_(y), month_(m), day_(d) {}
void disBirth()
{
        cout << "year_:" << year_ << endl;
        cout << "month_:" << month_ << endl;
        cout << "day_:" << day_ << endl;
}
private:
int year_;
int month_;
int day_;
};

class Student
{

public:
Student(string sn, char cs, float fs) :name_(sn), sex_(cs), score_(fs) {}
void dis()
{
cout << "name:" << name_<<endl;
cout << "sex:" << sex_ << endl;
cout << "score:" << score_ << endl;
}
private:
string name_;
char sex_;
float score_;
};

class Graduate:public Student
{

public:
Graduate(string sn, char cs, float fs,float fda,int y,int m,int d)
:Student(sn,cs,fs), birth(y,m,d)
{
salary_ = fda;
}

void dump()   //上面对象使用了dis这里就不再使用dis
{
dis();
birth.disBirth();
cout << "salary" << salary_ << endl;
}
private:
float salary_;
Brithday birth;
};


int main()
{
Graduate g("zhangsan",'s',99,1000,1999,9,9);
