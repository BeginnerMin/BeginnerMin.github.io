拷贝构造

我们再次强调初始化与赋值的区别：

int a = 0; //初始化
a = 5; //赋值

发生时机

由己存在的对象，创建新对象。

也就是说新对象，不由构造器来构造，而是由拷贝构造器来完成。

拷贝构造器的格式是固定的。

拷贝构造器和构造器的地位是一样的。

语法定义

class 类名 
{ 
类名(const 类名 & another) 
拷贝构造体 
}



class A 
{ 
A(const A & another) 
{} 
}

由普通数值作参数完成的构造，调用构造器。

由同类对象做参数完成构造就是拷贝构造。

普通数值参数完成构造说明如下：

#include <iostream>
using namespace std;

{

}
private:
int year;
int month;
int day;
};

int main()
{
Data d(2016, 8, 8);   //普通数值构造

return 0;

编译器只是提供一种等位拷贝机制。那么我们可以在拷贝构造器里面进行自实现等位拷贝：

构造特性

拷贝构造规则

浅(shallow)/深(deep)拷贝

系统提供的默认拷贝构造器是一种浅拷贝，浅拷贝就是如果对象中不含有堆上的空间，全部都是栈上的空间。此时浅拷贝可以满足需求。也就不需要自实现。

系统提供默认的拷贝构造器，一经自实现系统将不再提供。但系统提供的默认拷贝构造器是等位拷贝，也就是通常意义上的浅(shallow)拷贝。

如果类中包含的数据元素全部在栈上，浅拷贝也可以满足需求的。但如果堆上的数据，此时浅拷贝不能满足需求，则需要自实现拷贝构造实现深拷贝。

那么浅拷贝不仅仅只是不能满足需求，并且会带来冲析构的问题，double_free。

浅拷贝示例 class Date

本例中典型的特点，就是数据全部在栈上，无论是否自实现拷贝构造，均可以达到拷贝的目的。

如果对象中含有堆上的空间，此时浅拷贝不能满足需求，需要自实现。

浅拷贝不能满足需求指的是浅拷贝会带来重析构，即double free

深拷贝示例 class DataStr

本例中典型的特点，就是数据包含在堆上的空间，必须要自实现拷贝构造，达到深拷贝的目的，不然则会造成重析构，double free 的现象。

我们下面给出没有包含堆上的空间，但是没有自实现而使用系统默认的拷贝构造的情况。

为什么会出现重析构的现象呢？

本质还是因为等位拷贝：

我们前面说过，拷贝的时候拷贝的内容不包含类成员函数，只包含类对象数据，那么上面例子拷贝的时候，只是把char* _str; 指针进行了拷贝，拷贝完成之后，两个指针指向了同一块堆空间，也就是说第一个char* _str; 析构一次，拷贝之后的char* _str; 再析构一次，就造成了重析构。那么系统此时提供的等位拷贝就有问题，所以就需要我们自实现。

#define _CRT_SECURE_NO_WARNINGS
#include <iostream>
#include <string.h>
using namespace std;


class DataStr
{
public:
    DataStr()
    {
    _str = new char[1024];
    strcpy(_str, "C++ is th best language in the world\n");
    }
    DataStr(const DataStr& another)
    {
        cout << "copy constractor" << endl;
        _str = new char[strlen(another._str) + 1];
        strcpy(_str,another._str);
    }
    ~DataStr()
    {
        delete[]_str;
    }
    void dos()
    {
        cout << _str << endl;
    }
private:
        char* _str;
};
int main()
{
    DataStr ds;
    ds.dos();
    DataStr ds2(ds);
    ds2.dos();
    return 0;
}

没有问题。

浅拷贝/深拷贝图解说明

同类之间无私处，异类之间有友元

意思就是说同类对象方法中进行传递参数，可以直接访问同类对象的私有成员，其他则不行。
关于友元在友元博客进行说明。
上面深拷贝代码中的主函数进行修改：

int main()
{
    DataStr ds;
    ds.dos();
    DataStr ds2(ds);
    ds2.dos();
    DataStr (ds);
    return 0;
}

上面代码中：

DataStr (ds);

会导致代码出现很多错误。

DataStr ds2(ds);是使用同一类的对象进行拷贝构造，由一个对象创建另一个对象，相当于是同类之间发生关系。所以同类之间传递对象是可以直接访问的，之间访问类内的私有成员。

同类对象方法中进行传参可以访问其私有成员。其他则不可以。
