引用(Reference &)

定义

变量名，本身是一段内存的引用，即别名(alias)。此处引入的引用，是为己有变量起一个别名。
例如我们的

int a = 100；

实际上就是编译器在内存里面给我们申请了4个字节，给里面的存放的是数字100，但是避免我们直接和内存打交道，所以给这4个字节起了一个别名用a取代，所以我们给a放100实际上就是给a指向4个字节里面放100。变量名a本身就是别名，本身就是内存引用。此处的引用是为已有的变量取别名。

对于已有变量取别名就是引用，

例如我们现在对于a取一个别名，引用的方式为：

int a = 100;
int &ra = a;

此时

ra和a是等价的。

我们进行代码演示：

#include <iostream>
using namespace std;
int main()
{
int a = 999;
int & ra = a;
cout <<"a ="<< a << endl;
cout << "ra = "<<ra << endl;
cout << "&a =" << &a << endl;
cout << "&ra = " << &ra << endl;
cout << "sizeof(a) =" << sizeof(a) << endl;
cout << "sizeof(ra) = " << sizeof(ra) << endl;
}

执行结果为：

我们可以看到a和ra对应的值和地址都是一样的，也就验证了我们前面所说的等价，就会一种别名关系。

引用规则

①、引用，是一种关系型声明，而非定义。不能独立存在，必须初始化，且与原类型保持一致，且不分配内存。

代码里面只有：

int a = 999;
int & ra;

上面出现错误，引用必须初始化，不能单独存在。

int a = 999;
 double & ra = a;

上面出现错误，引用类型与原类型不一致

②、声明关系，一经声明，不可变更。

int a = 999;
int b = 111;
int & ra = a;
int & ra = b;

上面的代码是不允许的。不允许重新声明。

③可对引用，再次引用。多次引用的结果，是某一变量具有多个别名，多个别名间是平行关系。

#include <iostream>
using namespace std;
int main()
{
int a = 999;
int & ra = a;
int & rb = a;
int & rc = rb;
cout <<"a ="<< a << endl;
cout << "ra = "<<ra << endl;
cout << "&a =" << &a << endl;
cout << "&ra = " << &ra << endl;
cout << "sizeof(a) =" << sizeof(a) << endl;
cout << "sizeof(ra) = " << sizeof(ra) << endl;
cout << "============================" << endl;
cout << "a =" << a << endl;
cout << "rb = " << rb << endl;
cout << "&a =" << &a << endl;
cout << "&rb = " << &rb << endl;
cout << "sizeof(a) =" << sizeof(a) << endl;
cout << "sizeof(rb) = " << sizeof(rb) << endl;
cout << "============================" << endl;
cout << "a =" << a << endl;
cout << "rc = " << rc << endl;
cout << "&a =" << &a << endl;
cout << "&rc = " << &rc << endl;
cout << "sizeof(a) =" << sizeof(a) << endl;
cout << "sizeof(rc) = " << sizeof(rc) << endl;
cout << "============================" << endl;
}

执行结果为：

我们可以对引用再次引用，也可以对于最开始定义的变量进行多次引用，我们在前面就已经提到过，开始的变量名a也是对于内存中4个空间的引用。
上面代码中所有的引用都是平行关系。

④辨别引用与其它，&符号前有数据类型时，是引用，其它皆为取地址或按位与。

引用应用

C++很少使用独立变量的引用，如果使用某一个变量，就直接使用它的原名，没有必要使用他的别名。

引用的真正目的，就是取代指针传参。

C++引入引用后，可以用引用解决的问题。避免用指针来解决。

例如我们实现数据交互，之前是通过指针这样实现的：

#include <iostream>
using namespace std;


void myswap(int* pa, int* pb)
{
int tmp = *pa;
*pa = *pb;
*pb = tmp;
}
int main()
{
int a = 3;
int b = 5;
cout << "a = " << a << endl;
cout << "b = " << b << endl;
myswap(&a, &b);
cout << "a = " << a << endl;
cout << "b = " << b << endl;
return 0;
}

运行结果为：

我们可以看到实现了交换。

我们通过引用实现：

void mySwap(int& ra, int& rb)
{
int tmp = ra;
ra = rb;
rb = tmp;
}
int main()
{
int a = 3;
int b = 5;
cout << "a = " << a << endl;
cout << "b = " << b << endl;
mySwap(a, b);
cout << "a = " << a << endl;
cout << "b = " << b << endl;
return 0;
}

执行结果为：

此处传递引用等价于传作用域，把一个变量以引用的方式传到另外一个作用域，就等价于扩展了该变量的作用域。

我们知道如果直接在main函数里面通过中间变量进行交换然后打印交换后的结果肯定是没有问题的。所以这里就等价于把作用域进行传递，或者拓展作用域。

引用从宏观上可以理解为扩展了变量的作用域，传参后，就像在本地解决问题一样。避免了传 n 级指针，解决 n-1 级指针的问题，即平级内解决问题。
