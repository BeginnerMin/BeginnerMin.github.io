我们再次说明说很重要的一点：只有完全等价才能使用引用。

指针数据交换

接下来我们再看一个例子实现平级操作：

我们先给出在C语言中不能实现的一种情况

先给出代码：

#include <iostream>
using namespace std; 

void mySwap (char *p,char *q)
{
    char *t = q;
    q = p;
    p = t;
}
int main()
{
    char *p = "china";
    char *q = "canada";

    cout<<"p = "<<p<<endl;    
    cout<<"q = "<<q<<endl;

    mySwap(p,q);

    cout<<"p = "<<p<<endl;    
    cout<<"q = "<<q<<endl;
    return 0;
}

我们知道C语言的知识，我们知道交换肯定是不能成功的。

但是我们说，平级解决不了的事情，上一级可以解决，我们对于代码进行修改：

#include <iostream>
using namespace std; 

void mySwap (char **p,char **q)
{
    char *t = *q;
    *q = *p;
    *p = t;
}
int main()
{
    char *p = "china";
    char *q = "canada";
    cout<<"p = "<<p<<endl;    
    cout<<"q = "<<q<<endl;
    mySwap(&p,&q);

    cout<<"p = "<<p<<endl;    
    cout<<"q = "<<q<<endl;
    return 0;
}

执行结果为：

这样就交换成功了，但是在C++里面们应该避免使用指针。
接下来我们 通过使用指针的引用进行说明。

允许对于指针取引用

我们下面给出一个直接定义指针的引用来说明指针是可以引用的。

#include <iostream>
using namespace std;
int main()
{
    int a = 666;
    int* p = &a;
    int*  & rp=  p;
    cout << *rp << endl;
    return 0;
}

执行结果为：

我们可以看到使用成功。

我们现在肯定知道在main函数中定义中间指针变量进行交换肯定是没有问题的。那么我们还是要利用引用的作用域扩充来实现交换，那么接下来我们说明指针的引用，首先我们的两个变量是指针类型，然后我们通过引用来实现数据交换：

#include <iostream>
using namespace std; 

void mySwap (char * & pra,char * & prb)
{
    char *t = pra;
    pra = prb;
    prb = t;
}
int main()
{
    char *p = "china";
    char *q = "canada";
    char * & rp = p;
    cout<<"p = "<<p<<endl;    
    cout<<"q = "<<q<<endl;
    mySwap(p,q);

    cout<<"p = "<<p<<endl;    
    cout<<"q = "<<q<<endl;
    return 0;
}

打印结果为：

那么我能就可以对于指针类型进行引用。并且在平级解决问题。

不允许引用指向引用的指针

引用的本质是对指针的包装，避免使用裸露的指针，C++对裸露的内存地址(指针)作了一次包装。

又取得了指针的优良特性。所以再对引用取地址，建立引用的指针没有意义。

#include <iostream>
using namespace std;
int main()
{
    int a;
    int* p = &a; //对于a的地址进行引用 
    int * &pr = p;  //对于指针p再次进行引用
    int & ra = a;//对于变量a进行引用 
    int & * rpa = &ra;   //错误  不允许引用指向引用的指针
    //ra是  int &  类型  对于ra取地址 就形成int & * rpa = &ra;
    return 0;
}

对于引用的指针类型。C++避免对引用再次开封

所以不允许使用指针指向引用。对于引用的指针类型 C++避免对应用再次开封。

能够使用指针的指针

指针的指针，即二级指针。C++为了避免 C 语言设计指针的”失误”，避免了引用的引用这种情况。由此，也避免了引用的引用的引用的…..的引用的情况，这种可穷递归的设计本身就是有问题的。

#include <iostream> 
using namespace std;


int main()
{
int* p;
int** pp = &p;
int*** ppp = &pp;
int**** pppp = &ppp;
return 0;
}

可以无限次的取下去，是没有问题的。

不能使用引用的引用

可以对引用再次去引用，并且都是平行关系

不能使用引用的引用

上面两句话的使用是不一样的。

#include <iostream>
using namespace std;
int main()
{
    int a;
    int & ra = a;
    int & rb = a;
    int & rb = a;
    return 0;
}

上面的引用ra rb rc 都是对于a取别名，是平行的关系。

#include <iostream>
using namespace std;
int main()
{
    int a;
    int & ra = a;
    int &  & rra = ra;   //对引用的类型再次去引用是不允许的。
    return 0;
}

这里C++直接不允许引用的以用直接就不会有多层引用的使用。因为引用就是平级解决问题，平级可以解决没有必要去上一级。

有指针的数组，没有引用的数组

数组名，本身是首元素的地址，使用指针数组就是定义一个指针类型的数组是没有问题的。若首元素是引用的话，数组名就成了引用的指针， 与前面讲的不符，即不存在引用的指针。

#include <iostream>
using namespace std;
int main()
{
    int a, b, c;
    int *p[] = {&a, & b, & c};   //   指针数组p及就是二级指针
    int & q[] = { a,b,c }; //  int &  * 错误 不允许使用指向引用的指针
    return 0;
}

p代表首元素的地址，首元素a的地址是int* 类型。 p就是int **类型。

引用数组q里面，首元素a是引用类型。

int & q ，q代表数组首元素的地址，那么q 就是引用的指针，及就是int & * q = &a; 前面说过，不允许引用指向引用的指针。

数组的引用

数组名，本质是一个常量指针，对于指针是可以取引用的，所以数组的引用是存在的。

#include <iostream> 
using namespace std;

int main()
{
int arr[10] = {1,2,3,4,5,6,7,8,9};
//如果把arr单独看的话就是一个指针
int* const & parr = arr;
for (int i = 0; i < 10; i++)
{
cout << parr[i] << endl;
}
//如果把arr看作整个数组
int(&rarr)[10] = arr; 
cout << "sizeof(rarr) =" <<sizeof(rarr)<< "sizeof(arr) =" <<sizeof(arr) << endl;
return 0;
}
