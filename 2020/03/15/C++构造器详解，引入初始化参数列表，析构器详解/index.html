init 的初始化

前面我们封装 Date/Stack/List，虽然表达的意义各不相同，但是均会涉及到从类型到对象的初始化，即 init()初始化函数。 init()函数给予了对象的一个初始状态。这样一个初始状态对于后序的操作意义是重大的。 就像不论是什么样的结构必须进行了初始化才有后续操作。

#include <iostream>
using namespace std;

struct Data 
{
init();
};

struct Stack
{
init();
};

struct List
{
init();
};
int main()
{
return 0;
}

那么我们之前的struct里面必须进行初始化。那么初始化的过程是类到对象过程必须完成的一件事情，就是说必须有一个状态才能开始。

自动初始化

将上例中，init 函数，注释掉。换为了与类名相同的函数，该函数无返回，称为构造器，会在对象构造的时候，自动调用。 我们先进行生成自动调用的演示，然后使用构造器来替换声明。要说明的一点就是C++只是给我们提供一种初始化的机制，内容还是需要我们自己编写。

#include <iostream>
using namespace std;

class Stack
{
public:
Stack() {
cout << "xxxxxxxxxxxx" << endl;  //生成对象的时候调用
}
 int init(){
cout << "yyyyyyyyyyyy" << endl;  //调用的时候打印
return 0;
}

};


int main()
{
Stack s;
s.Init();
return 0;
}

自定义栈大小

上例中，构造器是无返回值的，但是可以有参数，通过传参的方式，就可以自定义栈的大小，这样的栈就可以作到量身定作了。

#include <iostream> 
#include <string.h> 
using namespace std; 
class Stack 
{
public: 
Stack(int size) 
{ 
top = 0; 
_size = size; 
space = new int[_size]{0}; 
}
bool isEmpty(); 
bool isFull(); 
void push(int data); 
int pop(); 
private: 
int *space; 
int top; 
int _size; 
};
bool Stack::isEmpty() 
{

return top == 0; 
}
bool Stack::isFull() 
{ 
return top == _size; 
}
void Stack::push(int data) 
{ 
space[top++] = data; 
}
int Stack::pop() 
{ 
return space[--top]; 
}

手动释放到自动销毁

上例中，我们通过传入参数的方式，动态的申请了堆内存的大小，既是堆内存，有申请必然会有释放的。 当然可以通过传统的方式来释放申请的空间，在本节，介绍一种特别用于释放资源的方式。析构函数，与类名同（前面需要带个~号），无参无返回。会在对象销毁时自动的被调用，我们可以将堆内存释放，放到其内。 

#include <iostream> 
#include <string.h> 
using namespace std; 
class Stack 
{
public: 
Stack(int size) 
{ 
top = 0; 
_size = size; 
space = new int[_size]{0}; 
}
~Stack() 
{ 
delete []space; 
}
bool isEmpty(); 
bool isFull(); 
void push(int data); 
int pop();
private: 
int *space; 
int top; 
int _size; 
};
bool Stack::isEmpty() 
{ 
return top == 0; 
}

构造器(Constructor)

发生时机

构造器，与类名相同，在对象生成的时候自动调用，那么我们就可以把初始化的内容直接放在构造器里面，无返回值。我们只把它理解为构造机制，或者叫构造器，不能理解为函数，可以有默认参数，有默认参数的时候优先使用默认参数，可以被重载。在类对象创建时(堆/栈对象)，自动调用，完成类对象的初始化。尤其是动态堆内存的申请。

语法

class 类名 
{ 
类名(形式参数) 
构造体 
}

class A 
{ 
A(形参) 
{} 
}

构造规则

1 与类名同，在对象创建时自动调用,完成初始化相关工作。

2 无返回值，可带参数，可以重载，可默认参数。

3 默认无参空实现体，一经自实现，系统默认提供的将不复存在。

4 无论重载还是默认参数，都应该把无参构造器包括在内。

自定义栈大小-重载构造器

我们现在增加需求：

例如上面的例子，在数组里面我们只需要10个空间，但是构造器直接给我提供了1024个空间。或者我要的更多的，但是最大只提供1024 所以不够灵活。

那么构造器在重载的时候不仅仅是加上参数，并且其他函数的内容也要进行修改。构造器重载代码如下：

#include <iostream>
#include <string.h>
using namespace std;


class Stack
{
public:
Stack()   //生成对象的时候调用
{   
top = 0;
space = new char[1024];
memset(space, 0, 1024);
SpaceSize = 1024;
}
//构造器重载
Stack(int size)   //生成对象的时候调用
{
top = 0;
space = new char[size];
memset(space, 0, size);
SpaceSize = size;
}
bool isFull();
bool isEmpty();
void push(char ch);
char pop();

private:
int top;
char* space;
int SpaceSize;
};




bool Stack::isFull()
{
return top == SpaceSize;  //修改判满条件
}

bool Stack::isEmpty()
{
return top == 0;
}

void Stack::push(char ch)
{
space[top++] = ch;
}
char Stack::pop()
{
return space[--top];
}


int main()
{
Stack s;//调用无参构造器
for (char ch = 'a'; !s.isFull()&&ch<='z'; ch++)
{
s.push(ch);
}
while (!s.isEmpty())
{
cout << s.pop();
}

cout <<endl<<"zzzzzzzzzzzzzzzzzzzzzzzzzzzzzzzz"<<endl;

//调用有参构造器
Stack s2(10);//传递构造器重载的参数

for (char ch = 'a'; !s2.isFull() && ch <= 'z'; ch++)
{
s2.push(ch);
}
while (!s2.isEmpty())
{
cout << s2.pop();
}
return 0;
}

但是我们既然能传递参数，那么会不会出现这样的问题，就是我们既然能够通过参数没有默认值进行栈空间大小的传递，那么上面无参构造器设置的1024是不是没有意义呢？

如果我们把无参的构造器去掉，那么s无法执行，但是如果我们把无参的构造器和有参没有默认值的构造器都去掉之后，就会产生默认的无参构造器，所以s可以运行但是不一定有结果，因为全靠人品，s2是有参数构造器，所以无法无法使用系统默认提供的无参构造器进行初始化。

如果我们自行实现构造器，实现的时候实现的是带参数没有默认值的构造器，那么s还是不能运行，只有s2能够运行。

如果我们自行实现构造器，实现的时候实现的是不带参数的构造器，那么s能运行，s2不能够运行，因为我们自实现的也是无参构造器。

因为我们一旦自实现构造器，系统默认提供的无参空体构造器就不再存在，不管构造器是否够符合类对象的运行要求。

初始化参数列表(initializelist)

初始化列表

C++提供了一种，不在构造体内初始化的方法，称为：初始化参数列表。一方面，提升了效率，另外一方面是功能的扩展。初始化参数列表，位于构造器声明与实现之间，“ ：”开始，各初始化成员用“ ，” 分开，其后的初始化成员置于“（）”以内。初始化列表只能在构造器中使用。

我们之前构造器中的：

   Stack(int size)   //生成对象的时候调用
{
top = 0;
space = new char[size];
memset(space, 0, size);
SpaceSize = size;
}

就可以改为：

Stack(int size = 1024)//生成对象的时候调用
:top(0),SpaceSize(size),space(new char[size]{0})  //初始化列表
{
}

上面就是使用初始化列表，完成private成员的初始化。并且只能初始化private成员，效率很高。

引入列表的初始化顺序

初始化的顺序和构造器中的赋值顺序无关，与private的定义顺序有关
我们通过代码进行演示说明：

#include <iostream>
#include <string.h>

using namespace std;

class A
{
public:
    A(char* str):_name(str),_len(strlen(_name.c_str()))
    {
    }

    void dis()
    {
        cout << _len << endl;
    }

private:
    int _len;
    string _name;
};

int main()
{
    char arr[6] = "china";
    A a(arr);
    a.dis();
    return 0;
}

上述代码是不能打印出来字符串的长度5。

A(char* str):_name(str),_len(strlen(_name.c_str()))

我们上面说明，初始化的顺序和 里面的赋值顺序是没有关系的，而是和private的声明顺序有关，及就是上面的代码上来一开始先算len那么name没有赋值，所以len就不会是5。

那么我们进行private的交换顺序：

#include <iostream>
#include <string.h>

using namespace std;

class A
{
public:
    A(char* str):_name(str),_len(strlen(_name.c_str()))
    {
    }

    void dis()
    {
        cout << _len << endl;
    }

private:
    string _name;
    int _len;

};

int main()
{
    char arr[6] = "china";
    A a(arr);
    a.dis();
    return 0;
}

打印结果为：

所以在初始化列表中也不要拿被初始化的成员去初始化其他成员。
初始化参数列表规则

1 ，此格式只能用于类的构造函数。

2， 初始化列表中的初始化顺序，与类内声明顺序有关，与列表中赋值顺序无关。

3，必须用此格式来初始化引用数据。

4， 必须用此格式来初始化非静态 t const 数据成员 (C++98) 。

第三点和第四点我们之后进行说明。

析构器(Destructor)

发生时机

以”~”开头与类名相同,无参无返回的函数，在类对象销毁时(栈/堆对象)，自动调用，完成对象的销毁。尤其是类中己申请的堆内存的释放。 

#include <iostream>

using namespace std;

struct Data
{
public:
Data(int y = 0, int m = 0, int d = 0)  //构造器
:year(y), month(m), day(d)
{}

~Data()
{
cout << "xxxxxxxxxxxxxxxxx" << endl;
}
private:
int year;
int month;
int day;
};
int main()
{
{
Data d;
}
cout << "yyyyyyyyyyyy" << endl;
Data* pd = new Data;
delete pd;
}

上面打印结果我们先进行解释说明：

第一行是创建的对象d在{}被使用完成之后打印的。

第二行是主函数进行打印。

第三行是通过pd创建对象在释放对象的时候打印的。

析构器的作用

在对象被销毁前被自动调用，用来处理清理工作。清理工作主要清理堆对象。
在对象销毁前：一种是栈上的对象一种是堆上的对象。
那么上面去示例，所有的空间都是在栈区申请，所以析构器存在的意义并不是很大，上面的析构器可以不用写，系统会提供一个无参空体析构器进行栈上数据的清理。

我们接下来举例一个需要的：
我们动态申请的空间，就需要内存释放;
栈对象在堆内存申请了1024个字节，在对象执行完成之后会释放1024个字节，构造机制和析构机制是就是系统给我们提供的自动调用的过程，里面的内容还是需要我们自行编写。
那么对于栈对象和堆对象都是一样的，在对象执行完成之后都需要调用析构器进行空间释放。
代码示例：

#include <iostream>
#include <string.h>
using namespace std;


class Stack
{
public:
Stack(int size = 1024)//生成对象的时候调用
:top(0), SpaceSize(size), space(new char[size] {0})  //初始化列表
{

}

bool isFull();
bool isEmpty();
void push(char ch);
char pop();
~Stack()
{
cout << "xxxxxxxxxxxxx"<<endl;
delete[]space;
}

private:
int top;
char* space;
int SpaceSize;
};




bool Stack::isFull()
{
return top == SpaceSize;  //修改判满条件
}

bool Stack::isEmpty()
{
return top == 0;
}

void Stack::push(char ch)
{
space[top++] = ch;
}
char Stack::pop()
{
return space[--top];
}


int main()
{
{
Stack s;   //栈对象
}

Stack* ps = new Stack(1000);  //堆对象
delete ps;
return 0;
}

所以析构器也并不是所有的时候都需要，在没有堆空间的对象中，使用默认，在对象有堆空间的情形下必须要有，析构器会在对象销毁的时候自动调用。
