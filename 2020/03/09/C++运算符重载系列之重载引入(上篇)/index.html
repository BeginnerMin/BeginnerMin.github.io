说明

从这篇博客开始，我会用连续的几篇博客对于C++运算符重载进行详细说明。

对于基本数据类型，比如 int 类型，我们对其进行加、减、乘、除运算，是语言层面
己经设置好的。

但是对于结构类型，比如 struct Complex 类型的对象是否也可以直接进行加、减、乘、
除运算呢，答案是否定的。

要想实现自定义类型加、减、乘、除运算，怎么操作呢，对加、减、乘、除运算符进
行重载，也就是本博客要说明的内容，运算符重载的本质是函数重载。

函数名为operator x（后面的x表示运算符）

重载引入

语法格式

重载函数的一般格式如下，operator 是关键字 ，后面跟的是运算符，operator+运算
符构成了新的函数名，此时我们说该运算符被重载了。

返值类型 r operator 运算符名称(形参列表)
{
重载实体;
}

友元重载

我们在说明友元的时候已经说明过友元重载，那么在这里我们直接使用。

#include <iostream>

using namespace std;

class Comlpex
{
    friend Comlpex operator+(Comlpex& c1, Comlpex& c2);
public:
    Comlpex(double r = 0, double i = 0) :real(r), image(i) {}

    void dumpFormat()
    {
        cout << "(" << this->real << "," << this->image << ")"<<endl;
    }

private:
    double real;
    double image;
};


Comlpex operator+(Comlpex& c1, Comlpex& c2)
{
    Comlpex c;
    c.real = c1.real + c2.real;
    c.image = c1.image + c2.image;
    return c;
}

int main()
{
    Comlpex c1(3, 4);
    c1.dumpFormat();
    Comlpex c2(5, 6);
    c2.dumpFormat();
    Comlpex sum = c1 + c2;
    sum.dumpFormat();
    return 0;
}

运行结果为：

及就是通过友元的方式实现了Comlpex对象相加。
但凡涉及到自定义类型想要跟普通数据类型一样的操作体验，就需要重载。
简单理解友元重载就是一个全局函数实现重载。

成员重载

友元重载到成员重载，少了一个参数。少的参数，即左操作数，也就是成员函数中默认的
this 指针。

#include <iostream>

using namespace std;

class Comlpex
{
public:
    Comlpex(double r = 0, double i = 0) :real(r), image(i) {}

    void dumpFormat()
    {
        cout << "(" << this->real << "," << this->image << ")"<<endl;
    }

    Comlpex operator+(Comlpex & another)
    {
        Comlpex c;
        c.real = this->real + another.real;
        c.image = +this->image + another.image;
        return c;
    }

private:
    double real;
    double image;
};



int main()
{
    Comlpex c1(3, 4);
    c1.dumpFormat();
    Comlpex c2(5, 6);
    c2.dumpFormat();
    Comlpex s = c1 + c2;
    Comlpex s1 = c1.operator+(c2);
    s.dumpFormat();
    s1.dumpFormat();

    return 0;
}

运行结果为;

我们在计算的时候并不会改变对象的内容，所以在对于对象进行应用的时候也需要加上const

局部变量不能返回引用。

栈上的对象可以返回，栈上的引用不可以返回。

对比赋值运算符重载

我们来对比我们之前赋值运算符重载的实现：

#include <iostream>

using namespace std;

class Comlpex
{
public:
    Comlpex(double r = 0, double i = 0) :real(r), image(i) {}

    void dumpFormat()
    {
        cout << "(" << this->real << "," << this->image << ")"<<endl;
    }

    Comlpex & operator=(const Comlpex & another)
    {

        this->real  = another.real;
        this->image =  another.image;
        return  *this;
    }

private:
    double real;
    double image;
};



int main()
{
    Comlpex c1(3, 4);
    Comlpex c2(5, 6);
    c2 = c1;
    c1.dumpFormat();
    c2.dumpFormat();



    return 0;
}

运算结果为：

在我们的基本类型中可以有以下写法：

int x,y,z;
x = y = z;
(x = y) = z;

对于+

x+y+z;
x+(y+z);
(x+y)+z;

上面都没有问题
但是

(x+y) = z;

就会出现问题。
那么对于我们上面创建的对象

(x+y) = z;

是可以实现的：

Comlpex operator+(Comlpex & another)
    {
        Comlpex c;
        c.real = this->real + another.real;
        c.image = +this->image + another.image;
        return c;
    }

返回c的时候并没有主函数去进行接受，那么就会返回在main函数的匿名对象，那么对象x+y就是一个匿名对象，匿名对象被复制为s。我们知道给匿名对象进行复制之后匿名对象最终会消失。那么就没有甚名实际的意义了，所以为了结局这个问题我们在前面加上const为：

const Comlpex operator+(Comlpex & another)
    {
        Comlpex c;
        c.real = this->real + another.real;
        c.image = +this->image + another.image;
        return c;
    }

这个时候

(x+y) = z;

就会直接报错。

通过上面理解那么我们对于代码整体进行修改后为：

所以我们之后就要主要在代码编写完成之后考虑是否需要加上const

const 修饰返值

C++引入引用的概念后，表达式可以被赋值的现象就出现了，有的表达式可以被赋值，
有的表达式则不可以。

比如，int a; int b; int c; (a=b)= c; 是可以的。而，(a+b) =c ;则是不允许的。

重载的运算符是否会导致表达式可以被赋值应该以基础类型为准。

返回类型通过加 const 加以限定来实现。

并不是所有的表达式均可被赋值。
