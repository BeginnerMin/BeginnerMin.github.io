成员函数作谁的成员？友元？

引例：邮件发送

假设，我们有类 Sender 类和 Mail 类，实现发送邮件的功能。
Sender sender; Mail mail;
sender << mail;
sender左操作数，决定了operator<<为Sender的成员函数,
而mail决定了operator<<要作 Mail 类的友员。

#include <iostream>

using namespace std;

class Mail
{
    friend class Sender;
public:
    Mail(string t, string c) :title_(t), contents_(c) {}
private:
    string title_;
    string contents_;
};

class Sender
{
public:
Sender(string addr):addr_(addr) {}
Sender & operator<<(const Mail & mail)
    {
        cout << mail.title_ << endl;
        cout << mail.contents_ << endl;
return *this;
    }
private:
string addr_;
};



int main()
{
Sender sender("hello@163.com");
Mail mail("开会","关于年底分红的若干事项");
Mail mail2("通知", "关于取消本次会议的通知");
    sender << mail;//发送邮件sender.operator<<(mail);   operator<<(sender, mail);
    sender << mail2;
    return 0;
}

声明为谁的友元，就可以通过谁的对象访问谁的私有成员。

结论

非全局莫属 operator<< / >>

不可能通过增加 istream 或 ostream 成员的方式重载>>/<<，此时只能通过在自定义的类中增加友元函数的方式重载>>/<<。

语法格式

friend istream & operator>>(istream &,自定义类&);
friend ostream & operator<<(ostream &,自定义类&);

operator>>

operator<<

#include <iostream>

using namespace std;


class Complex
{
public:
    Complex(double r = 0,double i = 0):real_(r),image_(i){}
private:
    double real_;
    double image_;
};

int main()
{
    Complex c;
    cin >> c;
    cout << c;
    //上面操作是不允许的。
    return 0;
}

上面代码会报错：

cin>>c;就会被强制解释为：cin.operator>>(c);

cout<<c;就会被强制解释为：cout.operator<<(c);

上面的实现是不允许的，cin，cout是标准库给出的，上面的操作需要进入cin的类中重载我们自定义的类型，标准库是不允许随便改的。

那么只能是换一种方式：

operator>>( cin,c);
operator<<( cout,c);

cin和cout 作为参数传递。

代码演示：

#include <iostream>

using namespace std;


class Complex
{
public:
    Complex(double r = 0,double i = 0):real_(r),image_(i){}
    friend istream & operator>>(istream & ci, Complex & c)
    {
        ci >> c.real_;
        ci >> c.image_;
        return ci;
    }
    friend ostream & operator<<(ostream & co, Complex & c)
    {

        co << "(" << c.real_ << "," << c.image_ << ")" << endl;
        return co;
    }

private:
    double real_;
    double image_;
};

int main()
{
    Complex c, c1;
    cin >> c;
    cout << c;
    return 0;
}

    cin >> c >> c1;
