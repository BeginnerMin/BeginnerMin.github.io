常量

常量的表现形式

常量是程序中不可改变的量

常量以字面量，或者宏的方式出现。

常量主要用于赋值或参与计算,并且常量也是有类型的。

字面量形式

我们先简单来理解常量和变量

#include <stdio.h>
int main()
{
   int age = 30;
   age = age + 10;
    return 0;
}

上面代码中，age是变量，30或者10是字面常量。

宏定义形式

#include<stdio.h>
#define PI 3.1415926
int main()
{
    int radius = 2;
    printf("area = %f", radius * radius * PI);
    return 0;
}

上面代码中，PI是宏定义常量。

宏定义在预处理阶段替换。

接下来我们进行预处理操作：

gcc -E main.c -o main.i

打开mian.i文件

使用预处理定义常量，在之后所有出现宏明的地方都会被替换，修改代码中的宏明对应的值的时候只需要在宏定义的时候进行修改即可。

常量的类型

我们进行类型说明：

#include <stdio.h>
int main()
{
        int number = 60;
    float salary = 1234.56;
    char sex = 'm';
    char name[] = "xiaoming";
        return 0;
}

上面的 60 是整型常量

上面的 1234.56 是实型常量

上面的 m 是字符常量

上面的 xiaoming 是字符串常量

那么常量到底是整型中的哪一种类型？实型又是具体的哪一种类型？我们下面进行说明。

我们说0是一个整型常量，但是能不能具体到某种类型的呢？

我们可以通过计算大小来进行解决：

printf("sizeof (0) = %d",sizeof(0));
printf("sizeof (0.0) = %d",sizeof(0.0));

运行结果如下：

上面两个都是常量0

我们通过代码验证：

上面的0是int类型

上面的0.0是double类型

默认的整型常量是 int 类型，默认的实型是 double 类型。

我们也可以通过下面操作进行修改：

出现在数据后面的字母是直接对于数据类型及就是所占字节数大小进行限制。

printf("const sizeof(0.0) = %d\n",sizeof(0.0)); //double
printf("const sizeof(0.0f) = %d\n",sizeof(0.0f));//0.0F float
printf("const sizeof(0.0L) = %d\n",sizeof(0.0L));//0.0l long double
printf("const sizeof(0) = %d\n",sizeof(0)); //int
printf("const sizeof(0l) = %d\n",sizeof(0l));//long
printf("const sizeof(0ll) = %d\n",sizeof(0ll));//long long

接下来我们介绍一种情况：

char ch = -1;
short sh = -1;
int in = -1;
printf("%x\n", ch);
printf("%x\n", sh);
printf("%x\n", in);

运行结果为：

那么为什么会出现这样的结果呢？

第一个char类型一个字节，是八位二进制数 ，用十六进制表示应该是 FF

一个F表示4位二进制数字。

第二个short 占用两个字节 所占16位二进制数字 那么应该打印FF FF 。

第三个 int类型4个字节，所以占32位 应该是FF FF FF FF。

那么结果应该是：

ff

ffff

ffffffff

实际的结果本应该是这样 只不过这里进行了整型提升

意思就是char类型在运算的时候会转换为int

short类型在运算的时候会转换为int

所以我们正常打印应该是这样打印：

char ch = -1;
short sh = -1;
int in = -1;
printf("%hhx\n", ch);   //这里我们给出%hhx 表示只打印一个字节
printf("%hx\n", sh);    //这里我们给出%hx 表示只打印两个字节
printf("%x\n", in);        //这里我们给出%x 表示只打印四个字节

在Qt编译器运行结果为：

在vs2019编译器下面运行结果为：

上面打印结果不同平台会有差异

那么char 类型-1 在内存中表示也就是1111 1111 就是FF

short 类型 -1 在内存中表示也就是1111 1111 1111 1111就是FFFF

int 类型-1 在内存中也就是 1111 1111 1111 1111 1111 1111 1111 1111

就是 FFFFFFFF

一定要注意：
** 内存中保存数据时保存的是补码，而不是我们直接计算的对应的二进制表示，应该是取补码之后的二进制表示为-1。**

例如 char类型 -1

机器码为 1111 1111

取反为 0000 0000

+1为 0000 0001

最开始的符号位为 1 所以为负数

补码之后的结果是1 所以表示 -1

所以的物理内存如何存储并不重要，重要的在于我们如何去解释这段物理存储。

例如 1111 1111 char 类型就代表 -1 无符号类型就代表 255

常量类型

整型常量

整型常量通常有三种表现形式

十进制常量

198 0 -23 216L 213u 213LU

L或者l 表示长整型

除了表示0以外不以0开头，避免和八进制混淆

U或者u表示无符号数

八进制常量

以0开头的连续数字序列 序列中只能有 0~7这8个数字

例如： 045 -076 034212l 028711u

十六进制常量

以0x 或者0X开头 序列中只有 09 AF a ~ f 这些数字和字母

字母a,b,c,d,e,f分别对应10，11，12，13，14，15 大小写都可以。

实型常量

小数形式
float salary = 1234.56;
double number = 6.54321;

上面直接出现的字面浮点数表示实型常量。

我们在这里说明一点：

printf(“.0”);
printf(“0.”);
printf(“0.0”);   //三者等价

printf(“.5”);   printf(“0.5”);   //意义相同
printf(“5.”);  printf(“5.0”);    //意义相同
指数形式

指数形式及就是科学计数形式：以10为底的幂数

要求：

①：字母e或者E之前必须要有数字

②：字母e或者E后面的指数必须为整数

字母e或E的前后及数字之间不允许有空格

默认是double类型占8个字节

后缀为f 或F 表示float类型

后缀l或者L表示为long double类型。

例如：34.2f .5E3 4.5e0 34.2f 2.5E3L

字符常量

‘a’ ‘b’ ‘c’

字符串常量

例如”a” “abcdefg”

小结

最后我们强调一点：

常量和变量在内存所保存的位置是不一样的，在后续博客我们进行详细说明
