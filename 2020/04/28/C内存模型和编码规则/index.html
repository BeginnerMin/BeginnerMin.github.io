内存模型和计算机编码规则将持续在我们整个计算机的学习之中

学习补码的必要性

在说明了C语言变量命名规则之后，这篇博客应该C语言说明变量的类型，但是我们首先要先说明内存模型和计算机编码规则因为内存模型和编码规则将持续在我们整个计算机的学习之中，并且能够让我们更加深刻的理解变量的类型。

内存模型

物理基础

现在的个人电脑我们经常使用到8G内存或者16G内存，操作系统有32位和64位，CPU也分64位和32位。

对于所有内存硬件来说，内存是线性的(物理硬件基础)，我们这篇博客以32位机来说明。

计算机内存的最小单元是bit 及就是一位二进制。

内存最小的寻址单位是字节。

也就是说以字节为单位分配地址。

我们在内存中读写数据就需要知道每一个内存中字节的地址。

下图中左边的16进制表示内存的地址，右边表示每一个字节的模型，一个字节里面有8位二进制数。

32位机能释放地址？

那么32位机也就是地址总线有32根

我们一共能够放下2^32个地址

2^32 = 2^10 x 2^10 x 2^10 x 2^ 10 x 2^2

2^32 = 4G 个地址

所以32位机最多释放4G内存个地址。

也就是说32位机的寻址能力是4G。

那么也就是说32位机只能装4G的内存，但是软件的内存开销，4个G是远远不够的。

即使是32位机并且插入了4G的内存也不会识别到4G的大小，一般到3G多将近4G，原因是统一编址的情况下网卡和显卡等其他设备都需要在内存中占据一部分内存空间。 所以即使是插入4G的物理内存也识别不到4G的内存。

由于软件的开销需要大量内存，4G是远远不够的，32位机不够用，所以我们现在更多的使用64位机。

2^64已经是非常大的数字，完全够我们使用。

64位机的分析方法和32位机相同。

CPU读写内存

那么接下来我们对于地址总线的进行说明：

CPU 在运作时，读写内存数据就需要先要指定内存储单元的地址。就是要确定读写哪段数据。然后指定是读操作还是写操作，最后根据指定的操作对于内存中的数据进行读操作或者写操作。

即要明确三件事。

①存储单元的地址（地址信息）

②操作的选择，读操作 or 写操作 （控制信息）

③ 读写的数据 （数据信息）

在这里，我们给出计算机导论的一些基础知识，方便我们之后的理解：

地址总线

其中 CUP 通过地址总线要寻址到指定存储单元。可见地址总线上能传送多少个不同的信息，CPU就可以对多少个存储单元进行寻址。

有 10 根地址总线，就能传送 10 位二进制数据，也就是2^10 。最小位 0，最大为 1023。也就是说n位地址总线可以生成2^n个地址。

CPU 地址总线的宽带决定了 CPU 的寻址能力。

数据总线

CPU 与内存或其他器件之间进行数据发送是通过数据总线来进行的。8 根数据总线一 次可以传送 8 位二进制数据就是1个字节数据。16 根数据总线一次可以传送 16位二进制数据及就是2个字节数据。 32位总线一次就可以传送32位二进制数据及就是4个字节数据。

数据总线的宽度决定了 CPU 和外界的数据传输速度。

控制总线

CPU 对外部部件的控制时通过控制总线来进行的。 控制总线是个总称，控制总线是由不同的控制线来集合的。有多少根控制总线，就意味着 CPU 提供了对外部器件的多少种控制。
例如：3位控制总线就有8钟控制，32位控制总线就有4G种控制。

控制总线的宽带决定了 CPU 对外部部件的控制能力。

读写流程

我们通过一个简单的图来描述三种总线的关系并且表示出CPU和内存之间的内部关系：

读取流程(从地址为3的内存单元读取数据)

①：CPU通过地址线将地址信息3发出。

②：CPU通过控制总线发出读内存指令，选中存储器芯片，并通知它，将要从中读取数据。

③：存储器将 3 号单元中的数据 8 通过数据线送入 CPU 。

我们对于上面过程进行解释：

CPU首先放出地址线为3，那么就能够寻址到地址为3的内存空间，然后通过控制总线发送的控制信号是读指令，那么就会把内存地址为3的数据8放到数据总线上，CPU就能够拿到数值。

写入流程(将数据26 写入内存地址为3的单元)

① CPU通过地址线将地址信息3发出。

② CPU通过地址线发出内存写命令，选中存储器芯片，并通知它，要其写入数据。

③ CPU通过数据线将数据26送入内存3单元中。

上图中先释放地址总线，找到地址为3的内存空间，控制总线发出的控制信号为写，那么就把数据总线中的值26写入内存地址为3的内存空间。

地址总线越多寻址的范围就越大，数据总线越多传递数据的速率就越快。

所以我们需要知道地址总线以及CPU的寻址。

我们把32位机变成64位机的时候能够访问内存中的地址就会更多，然前提是CPU是64位CPU才能够发挥作用。

32位平台包括:32位操作系统32位CPU，32位编译器。

在之后的我们学习编程语言过程中的编译器也必须是32位，这样才能整体发挥和识别。

上面我们了解到了内存模型之后，我们知道了CPU和内存之间如何访问，以及地址和内存空间相对应，也就是相当于我们知道了内存的物理模型和能够通过寻址找到每一个内存中的字节。

通常所说的32位机或者64位机指在单位时间内能处理字长为32位的二进制数据，也包括地址总线的范围。

计算机编码规则

现在我们已经知道了内存空间

但是一个字节内存空间能够装多大的数据呢？

我们讨论最小的单位一个字节的数据存储方式就可以解决所有的数据存储方式。

我们来看看一个字节到底能够装多大的数据。

大家知道 char类型就是一个字节 范围是[-128,127]

现在有一个问题是

-128和127能够取到吗？ 为什么不是-127~128呢？

在解决这个问题之前，首先说明一点：

一个字节的8位,我们先不去考虑符号的问题

从00000000到11111111总共有2^8及就是256种排列组合。

计算机的每一位只能是1或0

256种编码是物理基础，256种的1和0是确定的，那么关键是我们用什么样的方式来解释这256种组合，那么就涉及我们下面说明的编码。不同的编码规则会出现不同的数值范围。

编码

我们在这里表示一个字节可能出现的所有情况，上面所体现的是表示过程中的物理基础。

编码的过程就是我们对于物理基础的解释过程。

那么对于编码规则来说，我们有很多种。

以上给出来的是一个字节的不同的编码规则所体现出来的数字表示和数字范围。

仅仅按照编码规则来说，所有的编码规则都可以使用

但是为什么我们最后一定要用补码呢？

一个很重要的原因就是补码符合运算规则

从计算机出现到现在，所有的计算机只能做加法运算

具体说明如下：

对于第一种来说，比如我们要计算 1 + （-1）

那么就是 0000 0001加上 1000 0001按照编码方式结果就是1000 0010

那么1000 0010 对于上面编码来说对应的是-2

结果不是0 所以我们在编码规则的制定的时候不仅仅要考虑数据表示的问题，还要考虑到计算的的合理性。

接下来给出补码规则的计算方法 ：1 + （-1）

0000 0001+ 1111 1111 = 0000 0000

刚好溢出 等于 0 符合补码编码规则

设计的编码在进行计算之后的结果仍然符合编码规则

这才是运用补码的精妙之处。

上面两个计算只是给出来1和-1的计算

读者可以通过其他计算来验证补码在运算方面的合理性。

所以我们选择补码的原因：

① 可以表示数据

②可以进行合理的计算（重点）

接下来我们通过程序来打印数据的二进制形式来验证机器的补码存储：

#include  <stdio.h>
//打印一个数字的二进制
void disBinary(char ch)
{
    int i = 8;
    while (i--)
    {
        if ((1 << i) & ch)
            printf("1");
        else
        {
            printf("0");
        }
        if (i % 4 == 0)
            printf(" ");
    }
    putchar(10);
}
int main()
{
    char a = 0;
    disBinary(a);        //打印数据的二进制形式
    char b = 1;
    disBinary(b);        //打印数据的二进制形式
    char c = 2;
    disBinary(c);        //打印数据的二进制形式
    char d = 127;
    disBinary(d);        //打印数据的二进制形式
    char e = -128;
    disBinary(e);        //打印数据的二进制形式
    char f = -2;
    disBinary(f);        //打印数据的二进制形式
    char g = -1;
    disBinary(g);        //打印数据的二进制形式
    printf("");   
    return 0;
}

补码的思想来源

接下来我们讨论一个问题：当时的补码思想是怎么来的呢？

下来我们引入一个模的概念：

例如我们在调整钟表的时间的时候，我们既可以向前转动，也可以向后转动。也就是说每次时针转动12个小时就会回到原来的位置，那么对于编码也是一样，我们一个字节2 ^ 8再加1就会溢出编程0000 0000 也就是说数字经过2 ^ 8也会再次回到原来的数字。那么我们就称12小时，2^8为模。

时钟的计量范围是 0～11，模 =12。表示 n 位的计算机计量范围是 0～2 ^ n-1，模 =2 ^ n。 “模”实质上是计量器产生“溢出”的量，它的值在计量器上表示不出来，计量器上只能表示出模的余数。任何有模的计量器，均可化减法为加法运算。

例如：假设当前时针指向10点，而准确时间是 6 点，调整时间可有以下两种拨法：

一种是倒拨 4 小时，即：10-4=6；

另一种是顺拨 8 小时：10+8=12+6=6。

在以 12 模的系统中，加 8 和减 4 效果是一样的，因此凡是减 4 运算，都可以用 加 8 来代替。对“模”而言，8 和 4 互为补数。实际上以 12 模的系统中，11 和 1， 10 和 2，9 和 3，7 和 5，6 和 6 都有这个特性。共同的特点是两者相加等于模。

对于计算机，其概念和方法完全一样。n 位计算机，设 n=8，所能表示的最大数是11111111，若再加 1 成为 100000000(9 位），但因只有 8 位，最高位1自然丢失，又回了 00000000，所以8位二进制系统的模为 2^8。在这样的系统中减法问题也可以化成加法问题，只需把减数用相应的补数表示就可以了。

那么1-2在计算机内部就是1+ 2的补码。2的补数在模位8位二进制中及就是256 - 2 = 254

254的二进制表示就是0111 1110

对应负数就是1111 1110

1的二进制表示及就是0000 0001

0000 0001 + 1111 1110 = 1111 1111

1111 1111 的补码对应及就是-1

上面计算1-2的运算结果在计算机中是-1，也符合数学计算规则。

但是上面比较麻烦，所以直接使用在了编码当中，及就是补码的来历。

把补数用到计算机对数的处理上，就是补码。

小结

①原码和补码相互转化时，符号位参与运算，不需要单独处理。

②补码解决了正负0的表示方法。

③补码实现了计算机中的减法变加法，同样也适用于乘法和除法，数学逻辑方面的加减乘除都可以变成加法运算

④通过补码，计算机中所有的计算都可以转化为底层的加法计算。

一个8位的空间如果表示无符号数，表示0255 及就是02^8 - 1

如果表示有符号数就是：-128~127 及就是 - 2 ^ 7 ~ 2 ^ 7 - 1

最后我们打印出来一个字节表示的有符号数的补码形式：

#include <stdio.h>
//打印一个数字的二进制
void disBinary(char ch)
{
    int i = 8;
    while (i--)
    {
        if ((1 << i) & ch)
            printf("1");
        else
        {
            printf("0");
        }
        if (i % 4 == 0)
            printf(" ");
    }
    putchar(10);
}

int main()
{
    char ch;
    for (ch = -128; ch < 126; ch++)
    {
        printf("%d--> ", ch);
        disBinary(ch);
    }
    printf("%d--> ", 127); disBinary(127);
}
